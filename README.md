# Mini-Backup Engine (C++17, Boost.Asio, Zstd, SQLite)

Прототип высокопроизводительной системы многопоточного инкрементального резервного копирования для Linux. Проект разработан как демонстрация навыков системного программирования.

## 🛠 Технологический стек
Проект использует современные стандарты и библиотеки, критически важные для систем хранения данных:
* **C++17**: Активное использование **RAII**, умных указателей (`std::unique_ptr` с кастомными деструкторами) и лямбда-выражений.
* **Boost.Asio**: Реализация пула потоков (**Thread Pool**) для параллельной обработки задач через `boost::asio::post`.
* **Boost.Filesystem**: Надежная работа с путями файловой системы и метаданными в Linux.
* **Zstd (Zstandard)**: Интеграция через **Streaming API** для эффективного сжатия файлов любого размера с фиксированным потреблением ОЗУ.
* **SQLite3**: Инкрементальная логика на базе **Prepared Statements** для хранения хэш-сумм и метаданных файлов.
* **Boost.Test**: Модульное тестирование (Unit Testing) с использованием фикстур для проверки бизнес-логики.
* **Modern CMake**: Декларативное описание сборки и управление зависимостями.

## 🚀 Основные возможности
1. **Многопоточность**: Основной поток выполняет неблокирующий обход файловой системы, делегируя тяжелые операции (сжатие и I/O) воркерам в пуле.
2. **Инкрементальный бэкап**: Система вычисляет "быстрый хэш" (размер + дата модификации). Если файл не менялся, он пропускается (`SKIP`), что экономит ресурсы диска и CPU.
3. **Безопасность ресурсов**: Управление временем жизни контекстов Zstandard и дескрипторов SQLite реализовано через паттерн RAII, что исключает утечки памяти (Memory Leaks) и ресурсов (Statement Leaks).
4. **Масштабируемость**: Количество рабочих потоков адаптивно подстраивается под аппаратное обеспечение (`std::thread::hardware_concurrency`) или задается вручную.

## 🔨 Сборка и запуск

### Установка зависимостей (Ubuntu/Debian)
```bash
sudo apt-get update
sudo apt-get install build-essential cmake libboost-all-dev libzstd-dev libsqlite3-dev

Компиляция (Release-билд)
bash
mkdir build_release && cd build_release
cmake -DCMAKE_BUILD_TYPE=Release ..
cmake --build . -j$(nproc)

Использование
bash
./mini_backup <путь_источник> <путь_назначение> <файл_бд> [кол-во_потоков]

Пример:
bash
./mini_backup ./data ./backups ./metadata.db 4

🧪 Тестирование
Для запуска автоматизированных тестов (на базе Boost.Test) выполните:
bash
./run_tests

Тесты проверяют:
    Полноту цикла создания бэкапа.
    Корректность работы инкрементальной логики (пропуск неизмененных файлов).
    Целостность базы данных метаданных и синхронизацию пула потоков.

🏗 Архитектура проекта

    BackupEngine: Оркестратор. Управляет пулом потоков и логикой обхода директорий.
    StorageManager: Слой доступа к данным. Инкапсулирует работу с SQLite (RAII + Prepared Statements).
    Compressor: Функциональный модуль сжатия. Реализует Streaming API для Zstd.
    main.cpp: Точка входа, парсинг CLI-аргументов и глобальная обработка исключений.
